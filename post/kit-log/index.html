Kit Log

@Jerry · Jul 11, 2018 · 1 min read

kit 工具库提供了一个日志包。kit 认为日志库应该是最小接口和结构化的记录。

0x1 log包中的对象

kit logger 为一个接口类型，接口定义如下：

type Logger interface {
    Log(keyvals ...interface{}) error
}

实现这个接口都的类型都可以来记录日志。kit 提供了jsonfmt和logfmt 和空日志nop_logger 这个三个类型在实现logger接口的时候，根据自身目的的不同 ，格式化日志内容。日志需要输出， 所以在构造对应用类型的日志的时候，需要传实现io.writer接口的对象。下面为json logger的定义 和构造

type jsonLogger struct {
    io.Writer
}

// NewJSONLogger returns a Logger that encodes keyvals to the Writer as a
// single JSON object. Each log event produces no more than one call to
// w.Write. The passed Writer must be safe for concurrent use by multiple
// goroutines if the returned Logger will be used concurrently.
func NewJSONLogger(w io.Writer) Logger {
    return &jsonLogger{w}
}

输出的日志的时候，我们需要输出行信息以及时间信息，下面看看如何使用log包

0x02 使用

log 包使用时候，我们需要获得实现接口的对象,然后调用接口中的Log方法记录日志信息

var LogCaller = log.Caller(6)

logger = log.NewJSONLogger(log.NewSyncWriter(fd)) //注释1
logger = log.With(logger,
    "ts", log.DefaultTimestamp, //注释2
    "caller", LogCaller,  //注释3
)
logger = level.NewFilter(logger, level.AllowAll())//注释4
level.Info(logger).Log("aaa") //注释5
  • 注释1： 这里的fd可以是标准输出，标准错误，文件，或者自定义对象,只要是实现了io.writer接口的对象。经过包装后 支持同步写对象，其实里面就是加了个锁。将这个同步写对象再给NewJSONLogger构造一个日志json日志结构的对象
  • 注释2：log.With 函数将日志记录的时间写入到单行日志中
  • 注释3：log.With 函数将日志记录的行信息写入到单行日志中(这里需要记录的时间可以是静态的，也可以是动态的，详见kit doc)
  • 注释4：包装日志记录的级别判断

0x03 自定义日志接收对象

由上面可以知道，kit log的日志库提供了最小的调用接口和格式话日志的方法，但是日志的输出kit并没有提供。因为kit的日志定义是这样想的：

Collecting, shipping, and aggregating logs is the responsibility of the platform, not individual services. So, just make sure you’re writing logs to stdout/stderr, and let another component handle the rest.

但是还不知道哪个日志采集工具能从stdout中采集啊（有的话，请务必邮件告诉我）。所以下面参考别人的代码片段写了一个日志输出到文件中的类。支持天分割和文件大小分割:

 type logFile struct{
    curFile *os.File //当前写的文件
    fileName string //文件名
    sizeFlag bool //大小分割标志
    timeFlag bool //时间分割标志，按天分割
    filePath string //文件路径
    sizeValue int64 //文件分割大小
    todayDate string //当前记录时间
    msgQueue chan string //日志消息队列
    closed bool //文件关闭标志，这个参数，可要可不要，先留着
}
